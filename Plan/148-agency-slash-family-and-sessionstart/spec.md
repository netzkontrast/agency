# Spec 148 — `/agency` slash family + SessionStart Intent capture

## Why

The plugin manifest exposes exactly one slash command (`/help`). Spec
029 ships `agency_welcome` / `intent_bootstrap` as MCP tools, but a
human in Claude Code has no one-keystroke entry. Per the
plugin-development skill, a plugin's slash commands ARE its
conversational surface; per the managed-agents-onboarding pattern
(`claude-api` skill), first-touch should be a describe → configure →
confirm interview, with each turn captured as a graph Artefact so the
user feels creative while the engine captures everything (the GOALS.md
provenance moat).

## Done When

- [ ] **`commands/agency.md`** — `/agency [query]` routes to
      `mcp__agency__search` (tiered discovery, Spec 068) when given a
      query, else prints the live capability list via `agency_welcome`.
      Frontmatter: `description`, `usage`, `args` (per
      `developing-claude-code-plugins` skill).
- [ ] **`commands/agency-onboard.md`** — `/agency-onboard` runs the
      four-beat interview (describe → configure → confirm → capture).
      Each answer becomes an `Artefact(kind="onboarding-turn",
      beat: int, content: str)` with a `PRODUCES` edge from the
      bootstrapped Intent. The final beat calls
      `intent_bootstrap(purpose, deliverable, acceptance)` with
      values DERIVED from the captured Artefacts (never invents fields).
- [ ] **One `/agency-<skill>` shortcut per walkable skill** —
      generated by `install.py` from `<capability>.ontology.skills`
      (Spec 081 derive). A new skill auto-adds its command on `agency
      install`; a removed skill auto-removes its command. Derived from
      the live registry (Spec 149), never hand-authored.
- [ ] **SessionStart hook extension** — on first session in a repo
      with no open Intent (`agency.intent.list_open()` returns empty),
      emit a one-line prompt offering `/agency-onboard`. Non-blocking
      (respects Spec 062 idempotence); writes a `session-offer` Event
      (Spec 076) regardless of whether the user accepts.
- [ ] **`marketplace.json`** layered so `/plugin install agency` lands
      the slash family + the bare engine install in one step. Marketplace
      entry shape audited by Spec 177 (continuous reference compliance).
- [ ] **Acceptance invariant: provenance preservation** — when an
      `/agency-onboard` interview produces N turns, the resulting
      Intent has exactly N PRODUCES edges; querying via Spec 203
      (`graph_query`) recovers the full interview transcript. Lossless
      conversation → graph round-trip.
- [ ] **Failure mode: declined onboarding** — when the user says no,
      the hook does NOT re-prompt within the same repo (idempotence)
      and DOES record a `Reflection(scope="onboard-declined")` (Spec
      058) so the dogfood loop sees the rejection pattern.
- [ ] **TODO row + drift clean.**

## Worked example (Given/When/Then)

```text
Given:  Fresh agency install; no open Intent
When:   User opens a Claude Code session
Then:   SessionStart hook prints: "/agency-onboard to set up your work"
        AND records session-offer Event in the graph

Given:  User runs /agency-onboard
When:   They answer beat 1 ("I want to ship a novel chapter")
Then:   Artefact{kind:"onboarding-turn", beat:1, content:"...novel..."}
        is recorded; the Spec 147 Driver asks beat 2 (configure)

Given:  4-beat interview completed
When:   The fourth beat finishes
Then:   intent_bootstrap(purpose, deliverable, acceptance) called
        with values DERIVED from the four Artefacts
        AND the resulting Intent has 4 PRODUCES edges
        AND analyze.graph_query reconstructs the full transcript

Given:  /agency-onboard offered; user dismisses
When:   Next session opens
Then:   Hook does NOT re-prompt (idempotent per Spec 062)
        AND a Reflection(scope:"onboard-declined") is recorded
```

## Interconnects

- Anchors the **UX-onboarding chain** the charter declares.
- Spec 147 (AnthropicDriver) powers the conversational interview when
  the `[anthropic]` extra is present (managed-agents Outcome rubric);
  degrades to a static prompt list otherwise.
- Spec 149 (derived docs) generates the per-skill command files, so
  adding a skill auto-adds its `/agency-<skill>` shortcut.
- Spec 029 (welcome/bootstrap) is the MCP surface these commands wrap.
- Spec 076 (unified hook layer) is the SessionStart substrate.
- Spec 176 (sessionstart intent capture) builds on this — captured
  Intent seeds `AGENCY_INTENT` env (Spec 018).
- Spec 262 (managed-agents-onboarding-cap) wraps the interview as a
  first-class capability verb.
- Spec 177 (plugin-reference audit) gates the command frontmatter.
- Spec 203 (graph-query) recovers the interview transcript.

## Open questions

1. **One mega `/agency` with subcommands, or a flat family?**
   **Recommend**: flat family for discoverability (Claude Code lists
   each); `/agency` itself is the dispatcher. Matches the `claude-api`
   subcommand pattern.
2. **Pre-existing Intent — re-onboard or resume?** **Recommend**:
   resume — show the open Intent's purpose + a "/agency-onboard new"
   variant for explicit re-entry. Never silently supersede.
3. **Web vs CLI invocation differences.** Web Claude Code may not show
   slash commands the same way. **Recommend**: the SessionStart hook
   text fallback works in both; the slash commands are the
   one-keystroke shortcut on CLI.
4. **Command count budget.** If `/agency-<skill>` × 40 skills crowds
   the command palette, consolidate. **Recommend**: cap derived
   shortcuts at the top-10 most-invoked skills (derived rank, Spec
   149); the rest still reachable via `/agency <name>`.
