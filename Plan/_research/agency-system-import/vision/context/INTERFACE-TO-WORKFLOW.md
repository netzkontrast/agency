# Context Interface to Workflow

## 1. Stance
The `context` column views the `workflow` column as the state machine defining the "how" and "when" of the matrix. While `workflow` defines pipelines and gates, it is completely reliant on `context` for the shape of the data flowing through those pipelines (schemas) and the scaffolds that produce the end artefacts (templates). `context` expects `workflow` to strictly adhere to the defined storage vault topology and to accurately report ontology tags so the Graph Audit Trail correctly links artefacts to phases.

## 2. Surface
`context` exposes the following concrete interfaces to `workflow`:

- **Phase Validation Schema**: `workflow-cell.schema.json`
  - Validates `workflow/<row>/manifest.toml` and the structure of `phases/` and `gates/`.
- **Handoff Envelope Schema**: `tool_result.schema.json`
  - Defines the exact JSON Schema for the data traversing phase boundaries.
- **Scaffold Templates (Meta-Row)**:
  - `context/workflow/templates/agentic-cell.jinja`
  - `context/workflow/templates/workflow-cell.jinja`
  - `context/workflow/templates/context-cell.jinja`
- **Output Vault Resolvers**: `get_storage_path(row: str, vault: str) -> str`
  - Resolves standard paths (e.g., `storage/outputs/albums/`) based on `manifest.toml` `[storage]` entries.

## 3. Required of Target (`workflow`)
For the context column to maintain a coherent data model, `workflow` MUST provide:

- **Artefact Ontology Declaration**: `workflow` manifests MUST declare `artefacts_produced: list[str]` explicitly tagging the ontology types generated by a given phase.
- **Audit Trail Edge Emission**: When a workflow phase resolves and an artefact is finalized in the storage vault, `workflow` MUST emit a graph event mapping `Artefact -> SATISFIES_PHASE -> Phase`.
- **Template Isolation**: `workflow` MUST ONLY source output formatting templates from `context/<row>/templates/` or `context/_shared/templates/`. Inline templates in the `workflow` directory are forbidden.

## 4. Isomorphism Map
The `context` cell-shape fits tightly around `workflow`'s phase structure. `workflow`'s phases produce artefacts, and `context` provides the `storage/` directory and schemas to hold and validate them.
*Friction note:* `workflow` phases pass `ToolResult` envelopes sequentially, but complex cross-row dispatch can sometimes obscure the originating `SATISFIES_PHASE` edge if the `workflow` state machine doesn't pass the origin phase ID properly through the envelope.

  # Maps to Scenario #4: Typed envelope

    Given a workflow phase completes execution
    When the phase attempts to pass state to the next phase
    Then the handoff payload is strictly validated against `tool_result.schema.json`
    And a failure in validation triggers the phase's gate to block progression
