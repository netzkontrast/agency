## 1. The stack

## 2. What each spec changes about Path B

| Path B spec | Before Wave D | After Wave D |
|---|---|---|
| **111 manifest** | Indexes any markdown with frontmatter | Indexes only artefacts whose frontmatter validates against `l1-vault-core.schema.json` (Spec 122). Tag taxonomy formalises `domain:*`, `kind:*`, `topic:*`. |
| **112 describe** | Returns `{id, type, title, summary, tags, body_preview}` | Adds `neighbours: {incoming: [...], outgoing: [...]}` resolved via `graph_describe_node` (Spec 124) over `header-ontology.json` edges (Spec 122). |
| **113 watcher** | Emits `ChangeEvent(path)` on FS modify | Same event surface, but downstream Spec 123 hook validates ontology, Spec 124 incrementally upserts graph node + reloads CSR cache. |
| **111 BM25 search** | Fuzzy text match only | Coexists with `graph_cypher` (Spec 124) — text-search for keyword discovery, Cypher for structural traversal. Both surfaced via the same anchor triad. |

**Recommended wave sequencing:**

- **Wave D.0 (gate):** Path B (Specs **111 → 112 → 113**) must all have **implementations merged on Master** before 122 dispatches. Spec 122's frontmatter declares `depends_on: [111, 112, 113]` — 111 is the manifest format that bootstraps the type-system audit and graph initial-ingest; 112 is the anchor-triad whose `context_describe` consumes 122's edges via 124's `graph_describe_node`; 113 is the watcher whose `ChangeEvent` Spec 123's hook validates against 122's schemas. Dispatching 122 before any of the three is on Master will produce churn against incomplete prerequisites.
- **Wave D.1:** 122 (data shape). Single Jules session estimated. Spec ratifies PR #129's 12-type → 18-type extension and locks the L1 schema.
- **Wave D.2:** 123 (machinery). 2 Jules sessions estimated — handler port + hook wiring.
- **Wave D.3:** 124 (graph layer). 3 Jules sessions estimated — graphqlite install + schema mapping + algorithm tools + Path B integration.

122 + 123 are tightly coupled; 124 can theoretically ship in parallel with 123 once 122's schema is frozen, but practically wants 123's PostToolUse hook for incremental ingest.
