# Common Plugin Patterns

> Generic Claude Code plugin patterns. For agency-engine-specific patterns
> (capability, skill-walker, driver, substrate tool) see
> [`agency-patterns.md`](agency-patterns.md).

## Pattern: Simple plugin with one skill

**Use when:** focused plugin with documentation/reference material; no tools.

```
my-plugin/
├── .claude-plugin/
│   ├── plugin.json
│   └── marketplace.json
├── skills/
│   └── my-skill/
│       ├── SKILL.md
│       ├── scripts/           # optional
│       └── references/        # optional
└── README.md
```

Examples: `superpowers-developing-for-claude-code`.

Good for: teaching Claude a workflow (TDD, debugging, review), reusable knowledge bases.

## Pattern: MCP plugin with skill

**Use when:** the plugin ships a tool integration (MCP server) AND guidance on using it (skill).

```
my-plugin/
├── .claude-plugin/
│   └── plugin.json              # includes mcpServers config
├── skills/
│   └── using-the-tools/
│       └── SKILL.md
├── mcp/
│   └── dist/
│       └── index.js
└── README.md
```

**Key insight:** MCP provides *capability*, the skill provides *judgement*. The MCP server exposes `click()`; the skill teaches "await elements before clicking".

The agency plugin itself is this pattern at scale — see [`agency-patterns.md`](agency-patterns.md).

## Pattern: Command collection

**Use when:** multiple custom slash commands for project workflows.

```
my-plugin/
├── .claude-plugin/
│   └── plugin.json
├── commands/
│   ├── status.md
│   ├── deploy.md
│   └── rollback.md
└── README.md
```

Examples: `/deploy-staging`, `/incident-report`, `/code-review`.

## Pattern: Hook-enhanced workflow

**Use when:** automating actions in response to Claude's behavior.

```
my-plugin/
├── .claude-plugin/
│   └── plugin.json
├── hooks/
│   ├── hooks.json
│   └── run-hook.cmd             # polyglot wrapper, see polyglot-hooks.md
├── scripts/
│   ├── format-code.sh
│   └── run-linter.sh
└── README.md
```

Common hooks:
- `PostToolUse[Write|Edit]` → auto-format
- `SessionStart` → load project context
- `UserPromptSubmit` → enforce conventions
- `PreCompact` → save conversation summaries

**Warning:** hooks that block operations can disrupt flow. Make failure messages clear.

## Pattern: Full-featured plugin

**Use when:** comprehensive domain coverage with multiple integration points.

```
my-plugin/
├── .claude-plugin/
│   └── plugin.json
├── skills/
│   ├── main-workflow/
│   └── advanced-techniques/
├── commands/
├── hooks/
├── agents/
├── mcp/
│   └── dist/server.js
└── README.md
```

**Caution:** start simple. Most plugins don't need all components. Add when justified, not because the slot exists.

## Pattern: Skill with bundled resources

**Use when:** a skill needs reference material, scripts, or templates that would bloat SKILL.md.

```
skills/my-skill/
├── SKILL.md                    # main skill instructions
├── scripts/
│   ├── helper.py
│   └── generator.js
├── references/
│   ├── api-docs.md
│   ├── examples.md
│   └── cheatsheet.md
└── assets/
    ├── template.txt
    └── config-example.json
```

Resource use:
- SKILL.md tells Claude to "read references/api-docs.md for the complete API reference".
- Scripts can be executed via the Bash tool.
- Assets get copied/modified for output.

This is the pattern THIS skill uses.

## Choosing the right pattern

| Goal | Pattern |
|---|---|
| Teach Claude a process | Simple plugin with one skill |
| Add new tools + guidance | MCP plugin with skill |
| Provide project shortcuts | Command collection |
| Enforce conventions automatically | Hook-enhanced workflow |
| Comprehensive domain coverage | Full-featured plugin |
| Skill needs reference docs | Skill with bundled resources |

## Combining patterns

Patterns compose. The agency plugin combines:
- **MCP plugin with skill** (the FastMCP server + plugin-development skill)
- **Command collection** (slash commands generated by `agency.install`)
- **Skill with bundled resources** (this skill — SKILL.md + references/)
- **Full-featured** plus a domain-specific extension layer (capabilities self-register)

**Start simple, grow intentionally.** Add components when you have a clear reason, not because the slot exists.
