# Getting started with Agency

**Agency** is an installable Claude Code plugin and a small engine for building
agentic systems on one idea: **everything an agent does is a node in one
provenance graph, and code-mode is the only contract.** This guide gets you from
zero to running the engine and making your first call.

New to the model? Read [guide/concepts.md](guide/concepts.md) first.

## What you get

- An **engine** (`agency/`) — a FastMCP server over a bi-temporal graph
  (`graphqlite`), exposing exactly `search` / `get_schema` / `execute`.
- **10 core capabilities** out of the box — authoring (`plugin`,
  `skill_generator`, `develop`), orchestration (`delegate`, `subagent`, `gate`),
  dev-workflow VCS (`workspace`, `branch`), a remote agent (`jules`), and durable
  memory (`reflect`). Full list: [guide/capabilities.md](guide/capabilities.md).
- Installable **plugin** files at the repo root: `.claude-plugin/plugin.json`,
  `skills/`, `commands/` — generated by the engine itself.
- **Example extensions** in `examples/` (e.g. `examples/music.py`) showing how a
  domain capability plugs in without living in the core.

## Run it locally

```bash
python -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
pytest -q                      # 56 passing, on the real substrate (graphqlite + fastmcp)
```

## Install into Claude Code

This repository **is** the plugin (`.claude-plugin/plugin.json` is at the root).
Add this directory as a plugin (e.g. a marketplace entry whose `source` points
here), then `/agency:help` lists the capabilities. To regenerate the install
files from the live engine after adding a capability:

```bash
python -m agency.install
```

## Your first call (code-mode)

Everything serves an **Intent**, so bootstrap one, then `search` → `get_schema` →
`execute`:

```bash
python -m agency.cli --db g.db intent \
  --purpose "demo" --deliverable "a lint result" --acceptance "clean"
# -> {"intent_id": "intent:abcd1234"}

python -m agency.cli --db g.db search "lint a skill"

python -m agency.cli --db g.db execute --code '
  r = await call_tool("capability_plugin_lint_skill",
                      {"name": "fix-auth", "description": "Use when the auth test fails",
                       "intent_id": "intent:abcd1234"})
  return {"violations": len(r["violations"])}
'
```

The same contract is exposed three isomorphic ways — **MCP, Skills, and a bash
CLI** — so a bash-only agent like Jules is a first-class participant. Details in
[guide/usage.md](guide/usage.md).
