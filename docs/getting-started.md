# Getting started with Agency

**Agency** is an installable Claude Code plugin and a small engine for building
agentic systems on one idea: **everything an agent does is a node in one
provenance graph, and code-mode is the only contract.** This guide gets you from
zero to running the engine and making your first call.

New to the model? Read [guide/concepts.md](guide/concepts.md) first.

## What you get

- An **engine** (`agency/`) — a FastMCP server over a bi-temporal graph
  (`graphqlite`), exposing exactly `search` / `get_schema` / `execute`.
- **14 core capabilities** out of the box — authoring (`plugin`,
  `skill_generator`, `develop`), orchestration (`delegate`, `subagent`, `gate`),
  dev-workflow VCS (`workspace`, `branch`), a remote agent (`jules`), durable
  memory (`reflect`), code analysis (`analyze`), deep research (`research`), and
  doc rendering (`document`) + `dogfood` (graph-native observations). Full list:
  [guide/capabilities.md](guide/capabilities.md).
- Installable **plugin** files at the repo root: `.claude-plugin/plugin.json`,
  `.claude-plugin/marketplace.json`, `.mcp.json`, `skills/`, `commands/`, plus
  `hooks/` (a SessionStart polyglot wrapper that auto-installs the engine on
  first session per Specs 062/063/064) — all generated by the engine itself.
- **Example extensions** in `examples/` (e.g. `examples/music.py`) showing how a
  domain capability plugs in without living in the core.

## Run it locally

```bash
python -m venv .venv && . .venv/bin/activate
pip install -e ".[dev]"
python -m pytest -q -n auto -m "not e2e"  # 685 passing on the real substrate (graphqlite + fastmcp)
```

## Install into Claude Code

This repository **is** the plugin **and** its own single-plugin marketplace.
The committed install includes `.claude-plugin/plugin.json` (manifest +
`userConfig.jules_api_key`), `.claude-plugin/marketplace.json`, `.mcp.json`
(declares the Agency MCP server, launched via `bin/agency-mcp` in stdio mode
with `cwd: ${CLAUDE_PROJECT_DIR}` + `env_vars` passthrough),
`hooks/hooks.json` (SessionStart auto-install per Spec 062),
`skills/help/SKILL.md` + `skills/using-agency/SKILL.md` (the canonical entry
skill), and `commands/help.md` — all generated by the engine.

**From the GitHub marketplace** (in Claude Code Web or desktop):

```
/plugin marketplace add netzkontrast/agency
/plugin install agency@agency
```

That's it — the **SessionStart hook** (Spec 062) auto-runs `pipx install
--editable ${CLAUDE_PLUGIN_ROOT}` on the first session so `agency-mcp` lands
on PATH. The hook is idempotent on subsequent sessions. Three-step fallback
chain handles sandboxed environments:

1. **pipx** (canonical, Spec 055) — preferred.
2. **`pip install --user`** when pipx isn't available.
3. **`${CLAUDE_PROJECT_DIR}/.agency/.venv`** per-project venv (Spec 063) when
   neither global install path works. `bin/agency-mcp` knows to prefer this
   venv before falling through to PATH.

Claude Code prompts for your Jules API key (optional — only the `jules`
capability needs it; stored in your system keychain).

**Manual pipx install** (skips the hook — useful for local dev or scripting):

```bash
pipx install git+https://github.com/netzkontrast/agency@main
# …or for a local checkout:
pipx install --editable /path/to/agency
```

After install, three console-scripts land on PATH: `agency-mcp`
(MCP server), `agency` (CLI), `agency-doctor` (health check).

**Local-dev install (contributors):**

```bash
git clone https://github.com/netzkontrast/agency.git
cd agency
python -m venv .venv && . .venv/bin/activate
pip install -e ".[dev,analyze,recall]"
claude --plugin-dir .       # point Claude Code at this directory
```

On the next session the engine starts as an MCP server and `/agency:help`
lists the live capability set. **`using-agency`** (Spec 064) is the broad-
trigger entry skill — call it (or `agency_welcome` directly) at the start
of every agency-touching task to mint the Intent. To regenerate the install
files after adding a capability:

```bash
python -m agency.install     # regenerates .mcp.json + hooks + skills/help + commands
python -m agency.install --scaffold-db   # also creates .agency/ in the target repo
```

## Your first call (code-mode)

Everything serves an **Intent**, so bootstrap one, then `search` → `get_schema` →
`execute`:

```bash
python -m agency.cli --db g.db intent \
  --purpose "demo" --deliverable "a lint result" --acceptance "clean"
# -> {"intent_id": "intent:abcd1234"}

python -m agency.cli --db g.db search "lint a skill"

python -m agency.cli --db g.db execute --code '
  r = await call_tool("capability_plugin_lint_skill",
                      {"name": "fix-auth", "description": "Use when the auth test fails",
                       "intent_id": "intent:abcd1234"})
  return {"violations": len(r["violations"])}
'
```

The same contract is exposed three isomorphic ways — **MCP, Skills, and a bash
CLI** — so a bash-only agent like Jules is a first-class participant. Details in
[guide/usage.md](guide/usage.md).
