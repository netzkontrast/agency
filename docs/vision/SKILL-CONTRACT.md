# Agency skill contract — what every generated SKILL.md must teach

> Doctrine reference for Spec 031 §16. Every `skills/<capability>/SKILL.md`
> generated by `agency.install` (Spec 031 Phase 2) is a contract between
> the plugin and the calling agent. The contract has five obligations.
>
> Enforced by `plugin.lint_skill_doc` (Spec 031 §B) running in block mode
> at engine bootstrap (gated by `AGENCY_BOOTSTRAP_LINT=strict` per Spec 032
> §E unified env). Lint failure aborts the install — no bad SKILL.md ever
> reaches disk.

## 1. WHEN to call (trigger-first, not workflow-first)

The frontmatter `description` MUST start with `"Use when..."` and name 2–5
concrete triggering symptoms in the body's "When to use" section. A
description that summarizes the capability's workflow violates the contract
(Spec 023 / Spec 029 doctrine — also superpowers:writing-skills CSO rule).

Anti-pattern:
```yaml
description: Use to manage reflections.
```

Pattern:
```yaml
description: Use when recording a cross-session insight tagged by scope so
  a later session can search it back.
```

Rules `description-trigger-first` and `description-no-workflow-summary`
(and `overview-no-workflow-summary` for the body's overview section)
enforce this.

## 2. WHICH MCP verb (the wire surface)

The Verbs table MUST list every verb in the capability with:

- the verb name (kebab-cased `capability_<cap>_<verb>` on the wire);
- the role tag (`act` / `transform` / `effect`);
- a ≤ 60-char brief (from `parse_slices(docstring)["brief"]`);
- a relative-link reference (Tier A: separate file at
  `skills/<cap>/references/<verb>.md`; Tier B: anchor `#<verb>` within
  SKILL.md, for verbs that lack the Spec 016 Inputs:/Returns:/chain_next:
  docstring markers).

The agent MUST be able to pick the right verb from the table without
reading source.

## 3. WHAT the verb mutates (provenance trace)

For every `act` and `effect` verb, the Tier-A reference file MUST describe:

- which nodes get recorded (e.g., "records a `Reflection` node");
- which edges get linked (e.g., "`OBSERVED_DURING` + `SERVES` to intent");
- what the return delta looks like (the wire shape, not the in-sandbox
  `ToolResult` envelope — Spec 019 doctrine).

A `transform` verb's reference notes "pure: no graph or disk writes."

## 4. HOW to recover from common errors

The SKILL.md "Red flags" section MUST name 2–3 rationalizations the agent
might hit AND the counter:

```markdown
- "I already know what reflect.note does — skip the brief" → re-read the
  trigger list; you may be confusing reflect.note with reflect.search.
- "I'll skip intent_bootstrap, this is a quick read" → the SERVES guard
  will reject the call; mint an intent first.
```

Rules `red-flags-required` (≥ 1 red_flag if capability ships ≥ 3 verbs)
and `red-flags-format` (each item contains `→` or ` - ` as the
`<symptom> → <counter>` delimiter) enforce this.

## 5. WHERE to fetch heavy reference (progressive disclosure)

The SKILL.md body MUST link to per-verb references with markdown relative
links (`[details](references/note.md)`), never `@`-imports (which
force-load context). A fresh agent reading the SKILL.md gets ≤ 500 words
upfront and fetches reference files only on demand.

Tier classification (Spec 031 §5 Gherkin):

- Tier A: docstring carries all three Spec 016 markers (`Inputs:`,
  `Returns:`, `chain_next:`) AND each is non-empty. Terminal-by-design
  `chain_next: (terminal)` counts as A.
- Tier B: any marker missing or empty. Reference renders as an anchor
  inside SKILL.md rather than a separate file.

## Verification

`plugin.lint_skill_doc(cap_name, doc, verbs)` (Spec 031 §B) in block mode
+ `plugin.lint_skill(name, description)` (existing) together enforce all
nine rules in this contract:

- `description-trigger-first`
- `description-no-workflow-summary`
- `overview-no-workflow-summary`
- `triggers-named-symptoms`
- `triggers-count` (2–5)
- `example-uses-real-verb`
- `red-flags-required`
- `red-flags-format`
- `verb-briefs-resolve`

A capability whose SkillDoc fails any rule fails the install (the
generator never writes a bad SKILL.md file).
